# v0.2.0 (Beta) - Launcher UX & CI Integration 🚀

## 概要

本リリースでは、開発者体験（DX）の向上に焦点を当て、ランチャーの使い勝手を大幅に改善しました。また、CI/CD パイプラインを強化し、プロジェクトの安定性を高めました。

## 主な変更点

### 1. Launcher UX の刷新 (Go TUI) 🎨

- **実行順序の制御 (Step Dependency)**:
  - 誤操作を防ぐため、`Databases` -> `Crawler/API` -> `Frontend` という正しい起動順序を強制するロジックを実装しました。
  - 前提条件が満たされていないメニューは「Locked 🔒」としてグレーアウトされます。
- **視覚的フィードバック**:
  - 起動済みのサービスには「✅」マークを表示し、現在のステータスを一目で把握できるようにしました。
  - 画面下部に次に実行すべきアクションをガイドするメッセージエリアを追加しました。
- **コード品質向上**:
  - `main.go` の複雑なロジックをリファクタリングし、SonarQube の警告（Cognitive Complexity）を解消しました。

### 2. ドキュメント整備 📚

- **Launcher README (JA/EN)**:
  - 実行順序に関する注記を追加し、ユーザーが迷わないように配慮しました。
- **Main README**:
  - ロードマップを更新し、完了したタスク（CI/CD 構築など）をマークしました。
  - バージョンを **v0.2.0 (Beta)** に更新しました。

### 3. CI/CD パイプライン強化 🛠️

- **Go Build チェック**:
  - GitHub Actions (`ci.yml`) にランチャーのビルドチェックを追加しました。
  - これにより、ランチャーのコード変更がビルドを壊さないことを継続的に保証します。

## 今後の展望

- MySQL/PostgreSQL の自動インテグレーションテスト
- Docker イメージの最適化
