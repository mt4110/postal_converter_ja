# Alert Policy v0.9.0 (OPS-03)

最終更新: 2026-02-14 (JST)

## 1. 目的

`docs/SLO_SLI_v0_9_0.md` で定義した SLO/SLI を運用アラートに落とす。

## 2. 優先度定義

| Priority | 判定条件 | 一次対応期限 | 基本対応 |
| --- | --- | --- | --- |
| P1 | `ready` 失敗が10分継続、または5分平均5xxが3%以上 | 5分以内 | インシデント宣言、暫定復旧を最優先 |
| P2 | 5分平均5xxが1%以上、またはp95が300ms以上を15分継続 | 15分以内 | 影響範囲切り分け、復旧/緩和実施 |
| P3 | 更新ジョブ鮮度が12時間超、または異常傾向の早期警戒 | 当日中 | 監視強化、計画メンテ候補化 |

## 3. 通知経路

- P1: Incident チャンネル + On-call 即時通知
- P2: 運用チャンネル + 担当メンション
- P3: 日次運用レポートで集約

## 4. エスカレーション

| 経過時間 | P1 | P2 |
| --- | --- | --- |
| 0分 | 一次対応担当が受領・調査開始 | 一次対応担当が受領・調査開始 |
| 10分 | 復旧見込みなしなら platform 担当へ即時連絡 | 影響拡大時のみ platform 連絡 |
| 20分 | 事業側へ状況共有し暫定対処を先行 | P1 への格上げ判断 |

## 5. Runbook 紐付け

| 事象 | 参照Runbook |
| --- | --- |
| 更新失敗 | `docs/RUNBOOK_UPDATE_FAILURE_v0_9_0.md` |

## 6. ルール

- 受領時に `開始時刻 / 判定根拠 / 対応者 / 次アクション` を記録する。
- P1/P2 は復旧後24時間以内に振り返りを実施する。
- 閾値変更は `v0.9.xxx` Issue に根拠付きで記録する。
